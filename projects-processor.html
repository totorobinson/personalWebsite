A RISC Processor + Game

Our project is a ball balancing game that challenges players to keep a ball balanced on a platform by manipulating it with a joystick. The VGA display shows a timer and the high score for the current session, adding an element of competition to the game. To create the game, my team and I built a 16-bit RISC processor using Verilog, which we embedded in an FPGA.

We then developed a Python script that acts as an assembler and compiler, which reads a text file containing assembly instructions and outputs the corresponding machine code to a separate file. This script supports a wide range of instructions, including custom ones for VGA and UART interfacing, control flow, arithmetic, and loading values directly into memory during compile time. The script is also user-friendly, as it can be run from the command line.

Our software reads registers that hold joystick values sent from the Arduino via UART, which it uses to compute the motor values necessary for precise joystick control. To optimize the calculation, we use a combination of shift and add operations in place of multiplication, resulting in faster and more efficient execution. Finally, the motor values are transmitted back to the Arduino to control the motors. Because our FPGA did not have a UART module built into it, we had to make our own. The UART protocol we used to communicate with an Arduino for motor control and joystick input receives data asynchronously at 9600 Hz using a serial protocol that features one start bit, eight data bits, and one stop bit for each transmission.

Tools
Verilog, Python, Arduino
